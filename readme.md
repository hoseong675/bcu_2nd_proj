# 융합프로젝트: AI 기반 커스텀 PC 견적 추천 서비스 기획 및 아키텍처 정의서

본 문서는 학부 2학년 융합프로젝트(3인 팀)로 기획 중인 'AI 기반 커스텀 PC 견적 추천 애플리케이션'의 핵심 기능, 데이터 파이프라인, 그리고 라즈베리파이 5 기반의 인프라 아키텍처 설계를 정리한 명세서입니다.

---

## 1. 프로젝트 개요
* **프로젝트명:** AI 기반 커스텀 PC 견적 추천 서비스 (가칭)
* **개발 인원:** 학부생 2학년 3명
* **핵심 목표:** 사용자 맞춤형 설문 데이터를 기반으로, 로컬 DB의 호환성 검증을 거친 후, AI API를 활용하여 최적의 PC 견적 조합 3종(가성비, 안정성, 최고 성능)과 다채로운 자연어 추천 사유를 제공하는 시스템 구축

---

## 2. 핵심 기능 및 데이터 파이프라인

시스템은 단순 AI 의존형이 아닌, **'룰 기반(Rule-based) 필터링'**과 **'생성형 AI(LLM)'**를 결합하여 할루시네이션을 방지하고 정확도를 극대화합니다.

### 2.1단계: 사용자 입력 및 룰 기반 1차 필터링
1. **설문 입력:** 회원가입 시 하드웨어 지식/관심도를 입력받고, 로그인 후 구글 폼 형태로 정량적/정성적 요구사항을 수집합니다.
   * *예시:* 4K 해상도에서 특정 이펙트를 활용한 영상 편집 작업용, 예산 200만 원 내외 등
2. **소프트웨어 룰 필터링:** 목적에 맞는 핵심 칩셋(CPU, GPU)의 최소 가이드라인 표준을 백엔드에서 먼저 정의합니다.
3. **호환성 검사 (로컬 DB):** 최근 2년 내 출시된 최신 부품을 타겟으로 하여, CPU 소켓 규격, 메인보드 폼팩터, 파워서플라이 권장 출력, 쿨러 규격 등이 완벽하게 일치하는 부품들만 SQL  연산을 통해 후보군(Pool)으로 선별합니다.

### 2.2단계: 가격 데이터 캐싱 시스템
* **구현 방식:** 실시간 쇼핑 API 호출로 인한 네트워크 지연(UX 저하)을 방지하기 위해, 새벽 시간대 배치(Batch) 작업을 통해 다나와/네이버 쇼핑 등의 최신 가격 데이터를 로컬 DB의  테이블에 데일리로 업데이트(캐싱)합니다.
* **장점:** API Rate Limit 문제를 우회하고, 사용자에게 1초 이내의 빠른 검색 결과를 보장합니다.

### 2.3단계: AI API 연동 및 3종 커스텀 견적 생성
* **구현 방식:** 호환성이 검증되고 최신 가격이 매칭된 부품 후보군 리스트를 구조화된 JSON 데이터 형태로 AI API(OpenAI/Gemini)에 전달합니다. AI의  또는 을 사용하여 정형화된 응답을 수집합니다.
* **출력 결과:** 사용자의 예산 범위를 준수하는 3가지 최적의 조합과 정성적 이득을 반환합니다.
  1. **가성비 세팅:** 해당 스펙을 만족하는 제품 중 가장 저렴한 조합
  2. **안정성 세팅:** 스펙을 만족하며 예산 내에서 전원부, 방열, 쿨러, 파워 밸런스가 가장 뛰어난 조합
  3. **최고 성능 세팅:** 정해진 예산 한도 내에서 성능을 최대한 쥐어짜낸 하이엔드 조합

### 2.4단계: 3D 조립 가이드 (향후 확장 영역)
* **구현 전략:** 수천 가지 부품의 개별 3D 모델링은 현실적으로 불가능하므로, 사용자가 최종 선택한 견적의 **'폼팩터(표준 ATX, M-ATX, ITX 등)'를 기준**으로 삼습니다. 미리 구축해 둔 범용적인 부품 형태의 3D 조립 애니메이션(Unity WebGL 또는 Three.js 기반)을 가이드로 띄워주는 방식으로 구현하여 완성도를 높입니다.

## 3. 하드웨어 및 인프라 아키텍처

팀 프로젝트의 개발 및 실사용 데모를 위해 고성능 싱글보드 컴퓨터 기반의 홈 서버 아키텍처를 적용합니다.

### 3.1 하드웨어 스펙
* **메인 보드:** Raspberry Pi 5 (8GB RAM 모델)
* **스토리지:** 256GB NVMe M.2 SSD (PCIe 3.0 모드 활성화)
* **백엔드 프레임워크:** Java Spring Boot
* **데이터베이스:** MySQL

### 3.2 하드웨어 최적화 설정 (`/boot/firmware/config.txt`)
라즈베리파이 5의 기본 대역폭 제약을 해제하고 NVMe SSD의 성능을 최대로 끌어올리기 위해 아래 설정을 반드시 수반합니다.

    # PCIe Gen 3.0 모드 강제 활성화 (대역폭 약 800~900MB/s 확보)
    dtparam=pciex1_gen=3

* **도입 이유:** 데일리 가격 배치 쓰기(Write) 연산 및 대량의 부품 호환성 테이블 조인 읽기(Read) 연산 시 SD카드 대비 압도적인 속도를 보장하여 서버 병목을 원천 차단합니다.

### 3.3 가상 사설망(VPN) 인프라 구축
* **도입 기술:** **Tailscale (Mesh VPN)**
* **네트워크 구성:** 팀원 전체 기기와 라즈베리파이 서버를 Tailscale 가상 망으로 묶어 `100.x.x.x` 대역의 고정 사설 IP를 부여합니다.
* **장점:** 가정용 공유기의 포트포워딩(Port Forwarding)이나 복잡한 방화벽 설정 없이 외부망에 있는 팀원(프론트엔드, AI 파이프라인 개발자)이 안전한 암호화 터널을 통해 Spring Boot 및 MySQL에 다이렉트로 접속할 수 있습니다.
* **애플리케이션 바인딩 설정:**
  * Spring Boot (`application.yml`): `server.address`를 가상 IP로 제한
  * MySQL (`my.cnf`): `bind-address`를 가상 IP로 제한하여 보안성 극대화

## 4. 자원 할당 최적화 가이드 (8GB RAM 기준)

리눅스 OS의 안정적 운영과 가상 네트워크 오버헤드를 고려하여 총 8GB(8192MB) 자원을 다음과 같이 엄격하게 분할하여 OOM(Out of Memory) crash를 방지합니다.

1. **OS 및 가상 네트워크 기본 점유:** 약 800 MB (시스템 유지용)
2. **Spring Boot (JVM Heap Size):** 3,072 MB (`3GB`)
   * *실행 스크립트 옵션:* `java -Xms3072m -Xmx3072m -jar app.jar`
3. **MySQL (InnoDB Buffer Pool):** 2,048 MB (`2GB`)
   * *my.cnf 설정 항목:* `innodb_buffer_pool_size = 2048M`
4. **여유 공간 (Free Space):** 약 2,272 MB (여유롭고 안전한 상태 유지)
